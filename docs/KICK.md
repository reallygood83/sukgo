## 2. Why Now? — 왜 지금인가

### 시장 변화 3가지
1. **AI 구독 포화** — 사람들이 이미 Claude Pro / ChatGPT Plus / Gemini 등 한두 개는 결제 중. **추가 결제 거부감 최대**.
2. **CLI 르네상스** — Claude Code, OMC, aider 등으로 *"AI + 터미널"* 이 다시 매력적인 영역이 됨.
3. **결정 피로** — 정보 과잉 시대 → 사람들은 "더 잘 결정하기"에 진심.

### 타이밍의 의미
> **이 모든 게 동시에 맞아떨어진 첫 해가 2026년**.
> 1년 늦으면? 비슷한 도구들이 우후죽순 등장. 1년 빠르면? 인프라가 부족.
> **지금이 정확한 출발선**.

## 3. The Kick — 핵심 매력 3가지

> **"세 줄로 사람을 멈춰세우는 것이 KICK"**

### 🔥 KICK 1: "추가 결제 없는 AI CLI" ⭐ (메인 후크)
**기존 통념**: "AI CLI 만들려면 OpenRouter나 API 키 필요해."
**sukgo의 답**: "**아니, 네가 이미 내고 있는 Claude Pro·ChatGPT Plus 구독, 그대로 활용한다.**"
- 시청자 반응: *"이거 합법이야? 어떻게 가능해?"* → 클릭
- 핵심 시연 모델: **Claude (Pro/Max)** 또는 **Codex (ChatGPT Plus)** — 가장 많은 사람이 가진 구독
- 보너스 멘트: *"무료로 쓰고 싶다면? Gemini CLI도 됩니다. 로컬에서 돌리고 싶다면? MLX/Ollama도."*

> ⚠️ **시연·썸네일 규칙**: 영상에서는 **Claude·ChatGPT 사용 장면**을 메인으로. MLX는 보너스로만.

### 🔥 KICK 2: "AI 시대, 터미널이 가장 빠르다"
**기존 통념**: "GUI가 친숙하고 쉬워."
**sukgo의 답**: "**회의 5분 전, ChatGPT 켜는 30초 vs 한 줄 명령 2초.**"
- 시청자 반응: *"맞네... 나도 매번 ChatGPT 켜기 귀찮았어"* → 공감

### 🔥 KICK 3: "AI가 답하지 않는다 — 너를 키운다"
**기존 통념**: "AI 챗봇이 답을 준다."
**sukgo의 답**: "**sukgo는 너의 사고를 단단하게 만든다. AI가 너를 트레이닝한다.**"
- 시청자 반응: *"이건 다르네. 단순 챗봇이 아니구나"* → 차별화 인지

## 6. "왜 굳이 CLI?" 답변 매뉴얼

> 영상 댓글·소개 자리에서 무조건 받는 질문. 답을 외워두자.

### 🥇 한 줄 답
> **"GUI는 클릭의 시대 도구. CLI는 자연어 + AI 시대의 가장 자연스러운 인터페이스."**

### 🥈 3가지 강점 (외워둘 것)
1. **속도** — *"회의 5분 전 30초 vs 2초. 사고를 안 끊는다."*
2. **통합** — *"git diff | sukgo review가 가능한가? CLI니까 가능하다."*
3. **주권** — *"내 고민 기록이 평생 옵시디언에 쌓인다. 챗봇 기록은 그쪽 서버에 갇힌다."*

### 🥉 흔한 반박 대응표

| 반박 | 답변 |
|------|------|
| "ChatGPT 웹앱이 편한데?" | "편하지만 느리고, 매번 프롬프트 새로 침. sukgo는 사고법 자체가 명령어." |
| "GUI/앱이 친숙해" | "친숙은 타겟이 아닌 분 기준. sukgo 타겟은 충성도 최고인 10~20%." |
| "Claude Code 슬래시면 되잖아?" | "본인은 그것도 좋아요. 하지만 친구는 Claude Code 안 써요. CLI는 Claude Code 없는 사람도 쓸 수 있는 보편 인터페이스." |
| "터미널 못 쓰는 사람은?" | "타겟이 아닙니다. 진입장벽이 곧 정체성 마케팅 (Vim/Obsidian 패턴)." |
| "그냥 Python 스크립트면?" | "맞아요. CLI = 옷 입힌 Python 스크립트. `pip install`로 누구나 설치 가능." |

## 7. 브랜드 아이덴티티

### 이름의 무게
- **sukgo** = 한국어 "숙고(熟考)"의 로마자 표기 = *푹 익혀서 생각하다*
- 가벼운 챗봇과 정반대 — **시간을 들여 깊이 사고하는 도구**
- 영문 표기로 통일 → **호환성·검색·글로벌 확장성** 모두 확보
- 의미는 한국어 어원에서 → **차분한 무게감** 유지

### 톤 & 매너
| 키워드 | 의미 | 안 할 것 |
|--------|------|---------|
| 차분 | 무게감 있는 카피 | 가볍고 귀여운 어투 |
| 명료 | 한 줄로 핵심 | 장황한 설명 |
| 신뢰 | 검증된 프레임워크 강조 | 신박한 신기술 자랑 |
| 한국적 | 한자·한국어 친숙 | 영어 남발 |

### 시각 언어
- **로고/타이틀**: `sukgo` (영문 소문자, 등폭 폰트)
- **컬러 팔레트**:
  - 메인: **종이 베이지** (#F5F1E8) — 전통적 사색
  - 강조: **청록 (Teal)** (#2C7A7B) — 명료함
  - 경고: **흙색 적색** (#9B2C2C) — 차분한 강조
- **이모지 시그니처**: ✨ (만든 사람 표시 전용)
- **타이포**: 등폭 폰트 (터미널 기본 존중)

### 보이스 예시

❌ *"안녕! 오늘 어떤 고민이 있어? 같이 풀어보자!"* (가벼움)

## 8. 크리에이터 포지셔닝 — "배움의 달인"

### 브랜드 시너지

```
배움의 달인 (YouTube)              sukgo (CLI)
        │                              │
        ↓                              ↓
   더 잘 배우는 법              더 잘 생각하는 법
        │                              │
        └──────────┬───────────────────┘
                   ↓
           "메타 인지" 큰 우산
        (배움 + 사고 = 같은 본질)
```

### 일관된 메시지
- **배움의 달인 채널 메시지**: *"누구나 더 잘 배울 수 있다"*
- **sukgo 메시지**: *"누구나 더 잘 생각할 수 있다"*
- → **하나의 철학, 두 가지 도구**

### CLI 안 브랜딩 위치
1. **시작 화면 하단**: `made by 배움의 달인 ✨`
2. **`sukgo --version`**: `sukgo 0.1.0 — by 배움의 달인 (https://youtube.com/...)`
3. **세션 저장 마크다운 footer**: `> Generated by sukgo · made by 배움의 달인`
4. **README 푸터**: 채널 링크 + 짧은 소개

### 신뢰 자산화
- 영상에서 sukgo 시연 → 시청자가 직접 설치 → 매일 사용 → "**배움의 달인 = 진짜 만드는 사람**" 인식
- 그냥 "정보 채널" 에서 → "**도구를 만드는 크리에이터**" 로 격상

#### 📹 2편. **만드는 과정 공개**

#### 📹 3편. **실전 사용 5가지**

- 길이: 10~12분
- 핵심 메시지: KICK 3번 (사고를 키우는 도구)

### 시즌 2: 사고 도구 깊이 파기 (도구당 1편 = 10편)

## 부록 A. 자주 받을 질문 (FAQ)

### Q. 무료인가요?
**A**: 네, sukgo 자체는 완전 무료(MIT). 다만 백엔드(Claude/Codex/Gemini)는 사용자가 가진 구독을 사용합니다. 로컬 MLX/Ollama 사용 시 완전 무료.

### Q. 데이터는 어디 저장되나요?
**A**: 전부 사용자 컴퓨터에. sukgo는 텔레메트리·분석 시스템이 없습니다.

### Q. Windows에서도 되나요?
**A**: 네 (MLX 빼고). Windows 사용자는 Claude/Codex/Gemini/Ollama 백엔드 사용 가능.

### Q. 영어도 됩니까?
**A**: v0.1은 한국어 우선. v0.5에서 영어 i18n 예정.

### Q. 회사에서 써도 되나요?
**A**: MIT 라이선스 — 상업 사용 가능. 단 백엔드 약관(Claude/OpenAI)은 별도 확인 필요.

### Q. 다른 사고 도구 추가 가능?
**A**: v0.5에서 플러그인 시스템 도입 예정. 그 전엔 PR 환영.
